# homelab-status

A tiny, dependency-free status page that aggregates the health of the homelab
services (the kube-prometheus-stack + Loki monitored here, plus the
[aria-backend](https://github.com/evince55/aria-backend)) into one dashboard.

- **Standard-library only** — `python3 homelab_status.py`, nothing to install.
- **Server-side polling** — the server polls each endpoint on an interval and
  caches the result, so the browser never makes cross-origin requests (no CORS).
- **Resilient** — each check runs under its own timeout and `try/except`; one
  slow or dead endpoint never blocks or crashes the others.

## Run

```bash
python3 homelab_status.py --port 8080 --config config.json
# open http://localhost:8080
```

- `GET /` — HTML dashboard (dark theme; auto-refreshes `/api/status` every 10s).
- `GET /api/status` — the aggregated status as JSON.

## Configure

`config.json` is a list of services. Each entry:

| field | meaning |
|-------|---------|
| `name` | display name |
| `url` | health URL to poll |
| `type` | `http_ok` (2xx/3xx = up) or `http_json` (parse body, check a field) |
| `expected_field` / `expected_value` | for `http_json`: the field and value that mean "up" |
| `timeout` | per-request timeout (seconds) |

Add a service by appending an entry; the poller reloads `config.json` each cycle.

Note: the URLs are internal (Tailscale / in-cluster DNS), so run this on a host
with network access to them (e.g. inside the cluster or on the Tailnet).
